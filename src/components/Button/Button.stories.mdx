# Button

Buttons are used to show options for actions within a context. They should help the user find the
most important actions within a given context and allow them to perform those actions. The contents
should clearly indicate what action will be triggered when clicked.

## Kinds

There are four main kinds of buttons, each with different implications to the user:

Primary buttons should designate the _default_ action, and is displayed in a lovely shade of green.
Use this when there's one action you expect to be used most frequently.

### Borderless Buttons

Boarderless button are not only borderless but also backgroundless. Although they may look like plain p element - they still cover the same area as
any other button would do (so surface area is retained and hover effects are still applicable). They present the least signal for call to action, and
this intention is designed. Borderless button are best when place next to other buttons which are not tertiary to remind user's conceptual model of
buttons actions. Developers should use atmost one per form, otherwise use secondary button instead. Example usage can be included in cancel form/close modal/etc.

## States

States are overrides beyond the default styles for a given button. They are used to indicate status
of the button, such as whether it is non-interactive or if it's doing something.

### Disabled State

When a button is disabled, it will not respond to user interaction. Additionally, it is displayed in
a greyed-out style, to indicate that it is not available to the user.

<Canvas>
  <Story
    name="State: Disabled"
    parameters={{ layout: 'centered' }}
    argTypes={{
      kind: { table: { disable: true } },
      size: {
        defaultValue: 'large',
      },
      extendedPadding: {
        defaultValue: 'padding-lg',
      },
      alternativeColor: { table: { disable: true } },
      hoverBehaviour: {
        defaultValue: 'bg-darken',
      },
      theme: {
        defaultValue: 'light',
      },
    }}
    args={{
      kind: 'primary',
      children: 'Create Account',
      loading: false,
    }}>
    {(args) => <Button {...args} disabled />}
  </Story>
</Canvas>

### Loading State

When a button is disabled, it will not respond to user interaction. Additionally, it is displayed in
a greyed-out style, to indicate that it is not available to the user.

## Compositions

While Buttons are often usable on their own, you'll often want to compose them with other components
to create a more complex UI.

### With Icon

Sometimes you want an icon on your button. You can just include the icon as a child of the button,
and if it's an SVG it will be automatically styled.

import { FaTrash } from 'react-icons/fa';

<Canvas>
  <Story
    name="With: Icon"
    parameters={{ layout: 'centered' }}
    args={{
      kind: 'primary',
      children: 'Delete Item',
      loading: false,
      disabled: false,
    }}>
    {({ children, ...args }) => (
      <Button {...args}>
        <FaTrash /> {children}
      </Button>
    )}
  </Story>
</Canvas>
